JavaScript Techniques

💡 Click any card to reveal the code pattern

  • 🌐

    DOM Manipulation

    Select, modify, and create elements dynamically using the browser's living document tree.

    const el = document.querySelector('.js-target'); el.textContent = 'Updated!';
  • 👂

    Event Listeners

    Wire the DOM's nerves — listen for clicks, keypresses, and user interactions using addEventListener.

    btn.addEventListener('click', () => { doSomething(); });
  • 🧠

    State Management

    Track UI state in JavaScript variables. When state changes, update the DOM to reflect it.

    let isDark = false; btn.onclick = () => { isDark = !isDark; render(isDark); };
  • 🎨

    Dynamic Classes

    Use classList.toggle() to switch visual states. JS handles logic, CSS handles the animation.

    menu.classList.toggle('is-open'); btn.classList.add('is-active');

Common Questions

FAQ — Accordion

An accordion is a classic JS pattern: click to toggle visibility. Uses classList.toggle('is-open').

  • The Document Object Model (DOM) is the browser's volatile, living map of your HTML. It translates static markup into a tree of JavaScript objects (nodes), exposing a programmatic API for JavaScript to interact with the page. It is rebuilt entirely each time the page reloads.

  • const signals immutable binding — you will never reassign this variable. When grabbing a DOM element with querySelector, the reference itself doesn't change (only the element's properties do), so const is always the right choice. Reserve let strictly for values that must mutate — like counters or state flags.

  • Using innerHTML to insert user-supplied data is a direct path to Cross-Site Scripting (XSS) vulnerabilities — an attacker can inject malicious <script> tags. Always use textContent for safe data injection, or document.createElement() with appendChild() to build complex, data-driven interfaces programmatically.

  • Professional code relies on a predictable contract between markup and logic. The js- prefix is used strictly as JavaScript hooks — never for CSS styling. The is- prefix defines visual state (e.g., .is-active, .is-hidden). This decoupling means a designer can restyle everything without breaking JS, and a developer can refactor JS without touching CSS.

  • Neither is universally "better" — they are tools for different scales. Imperative (Vanilla JS) means writing explicit step-by-step instructions to find and update DOM nodes. Declarative (React, Vue) means defining what the UI should look like based on state, and letting the framework update the DOM automatically. Mastering imperative Vanilla JS first gives you the foundation to understand why frameworks exist.
